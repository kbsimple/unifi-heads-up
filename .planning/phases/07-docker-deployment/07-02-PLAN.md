  <action>
    Create Dockerfile at the repo root. Follow the two-stage pattern from the official vercel/next.js/examples/with-docker Dockerfile, adapted to two stages (no separate deps caching stage). Use node:22-alpine for both stages per D-02.

    CRITICAL REQUIREMENTS (will cause silent failures if missed):
    1. `ENV HOSTNAME="0.0.0.0"` MUST be in the runner stage — without it the server binds to the container ID hostname and is unreachable from the LAN even with correct port mapping (confirmed pitfall in RESEARCH.md)
    2. COPY sequence in runner stage MUST be:
       a. `COPY --from=builder /app/public ./public`
       b. `RUN mkdir .next`
       c. `COPY --from=builder /app/.next/standalone ./`  ← flattens to /app/server.js
       d. `COPY --from=builder /app/.next/static ./.next/static`
    3. NO `ENV UNIFI_HOST`, `ENV UNIFI_API_KEY`, or `ENV SESSION_SECRET` directives — per D-05

    Include `USER node` in the runner stage (non-root, security hygiene — the `node` user exists in official Node.js Alpine images).

    Write this exact content:

    ```dockerfile
    # syntax=docker/dockerfile:1

    # ---- Builder stage ----
    FROM node:22-alpine AS builder
    WORKDIR /app

    # Install dependencies (bcryptjs is pure JS — no apk build tools needed)
    COPY package.json package-lock.json ./
    RUN npm ci

    # Copy source and build
    COPY . .
    ENV NODE_ENV=production
    RUN npm run build

    # ---- Runner stage ----
    FROM node:22-alpine AS runner
    WORKDIR /app

    # HOSTNAME=0.0.0.0 is required — without it the standalone server binds to the
    # container ID hostname and is unreachable from the LAN even with port mapping.
    ENV NODE_ENV=production
    ENV PORT=3000
    ENV HOSTNAME="0.0.0.0"

    # Run as non-root user (node user exists in official Node.js Alpine images)
    USER node

    # Copy public assets
    COPY --from=builder --chown=node:node /app/public ./public

    # Prepare .next directory
    RUN mkdir .next

    # Copy standalone server and all inlined dependencies
    COPY --from=builder --chown=node:node /app/.next/standalone ./

    # Copy static assets — not included in standalone automatically
    COPY --from=builder --chown=node:node /app/.next/static ./.next/static

    EXPOSE 3000

    CMD ["node", "server.js"]
    ```

    Note: `next build` in the builder stage will succeed without UNIFI_HOST set because baseUrl() reads process.env.UNIFI_HOST inside its function body (not at module init) — confirmed in STATE.md.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker image layers | Secrets must not appear in any layer — visible via `docker history` |
| env_file → container | .env.prod passes secrets at runtime; must never be committed to git |
| LAN → container:3000 | Unauthenticated network access to the app port |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Information Disclosure | Dockerfile ENV directives | mitigate | No ENV directives for UNIFI_HOST, UNIFI_API_KEY, or SESSION_SECRET in Dockerfile (per D-05). Verify with `grep -E 'ENV (UNIFI|SESSION)' Dockerfile` returns empty. |
| T-07-02-02 | Information Disclosure | .env.prod committed to git | mitigate | .env* wildcard in .gitignore catches .env.prod. Only .env.prod.example (placeholder values) is committed. |
| T-07-02-03 | Elevation of Privilege | Container running as root | mitigate | Runner stage uses `USER node` — non-root execution. node user exists in node:22-alpine official image. |
| T-07-02-04 | Denial of Service | Container not recovering after host reboot | accept | `restart: unless-stopped` in docker-compose.yml ensures automatic recovery. Acceptable for LAN household use. |
</threat_model>
