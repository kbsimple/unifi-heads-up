## Interaction Patterns

### Navigation Pattern

**Top Navigation Tabs:**
- Three tabs: "Dashboard", "Firewall", and "Groups"
- Active tab: `text-sky-600 border-b-2 border-sky-600`
- Inactive tab: `text-zinc-400 hover:text-zinc-100`

```
[Dashboard] [Firewall] [Groups]
     ^                    ^
  existing              NEW (Phase 4)
```

**Source:** D-01, D-02 from CONTEXT.md

### Group Card Pattern

```
+---------------------------------------------------------------+
| [Group Name] (3 devices)              [Badge: High] [X icon] |
+---------------------------------------------------------------+
| [DeviceChip] [DeviceChip] [DeviceChip]                         |
| [+ Add Devices]                                               |
+---------------------------------------------------------------+
```

- Group name: Left-aligned, `font-medium text-zinc-100`
- Device count: `text-sm text-zinc-500` in parentheses
- Traffic badge: Badge variant based on aggregated status
  - High: `variant="default"` (sky-600)
  - Medium: `variant="default"` with yellow color
  - Low: `variant="default"` with green color
  - Idle: `variant="secondary"` (zinc-700)
- Delete button: `text-zinc-500 hover:text-red-400` with X icon
- Device chips: Pills with device name and remove X

**Card styling:** Same as Phase 2 device cards
- `bg-zinc-900 border-zinc-800 rounded-lg`
- Padding: `p-4`

**Source:** D-03, D-06, D-07, D-10, D-11 from CONTEXT.md

### Device Chip Pattern

```
+----------------------------------+
| [Device Name] [X]                |
+----------------------------------+
```

- Pill shape: `rounded-full bg-zinc-800 px-3 py-1`
- Device name: `text-sm text-zinc-300`
- Remove button: `text-zinc-500 hover:text-red-400`

**Source:** D-07 from CONTEXT.md

### Group Creation Modal Pattern

```
+-------------------------------------------------------+
| Create Device Group                              [X]  |
| Enter a name and select devices to add to this group |
+-------------------------------------------------------+
| Group Name                                            |
| [_________________________]                           |
|                                                       |
| Select Devices                                        |
| [ ] Device 1                                          |
| [x] Device 2                                          |
| [ ] Device 3                                          |
|                                                       |
|                        [Cancel] [Create Group]         |
+-------------------------------------------------------+
```

- Dialog trigger: Button with "Create Group" label
- Input: Group name (required, max 50 chars)
- Device selection: Checkbox list with device names
- Primary action: "Create Group" button (sky-600)
- Secondary action: "Cancel" button (zinc-700)

**Source:** D-04, D-05 from CONTEXT.md

### Traffic Chart Pattern

```
+---------------------------------------------------------------+
| Traffic (24 hours)                                            |
|                                                               |
|   Mbps                                                        |
|   10 |    ___/\___                                            |
|    5 |   /         \___                                       |
|    0 |__/                  \___                               |
|      +--------------------------------------------------      |
|      12am  4am  8am  12pm  4pm  8pm  12am                     |
+---------------------------------------------------------------+
```

- Chart type: AreaChart (Recharts)
- Y-axis: Bandwidth in Mbps
- X-axis: Time (hourly labels, 24 points)
- Fill: sky-600 gradient (opacity 30% at top, 0% at bottom)
- Stroke: sky-600 solid
- Tooltip: Shows exact value on hover

**Chart container styling:**
- `bg-zinc-900 border-zinc-800 rounded-lg`
- Height: 200px (per RESEARCH.md example)

**Source:** D-13, D-15, D-16 from CONTEXT.md

### Expandable History Panel Pattern

```
+---------------------------------------------------------------+

+---------------------------------------------------------------+
| Last active: 2m ago                                            |
|                                                               |
| [View History] ← click to expand                              |
+---------------------------------------------------------------+
|              ↑ EXPANDED STATE ↓                               |
+---------------------------------------------------------------+
| Traffic (last 24 hours)                                       |
| [AreaChart here]                                              |
+---------------------------------------------------------------+
```

- Trigger: "View History" text link
- Expand animation: Smooth height transition
- Chart appears inline below metadata

**Source:** D-13 from CONTEXT.md

## States

### Loading State
- Use `Skeleton` component from Phase 2
- Display 3 skeleton group cards while data initializes
- Same pattern as Phase 2 device loading

### Empty State (Groups)
- Icon: `FolderPlus` from lucide
- Heading: "No groups created yet"
- Body: "Create your first group to organize and monitor devices together."
- Action: "Create Group" button

### Empty State (History)
- Text: "No traffic history available yet. History accumulates during your session."
- Appears in expandable panel when no data collected yet

### Error State (localStorage)
- Toast notification: "Unable to save group. Please try again."
- Graceful fallback: Continue with in-memory state

### Error State (Group Creation)
- Toast notification: "Failed to create group. Please try again."
- Dialog remains open with error indication

### Delete Confirmation
- Dialog with confirmation
- Title: "Delete Group"
- Body: "Are you sure you want to delete \"{groupName}\"? This cannot be undone."
- Actions: "Cancel" (secondary), "Delete" (destructive variant)

**Source:** Patterns established in Phases 2 and 3
