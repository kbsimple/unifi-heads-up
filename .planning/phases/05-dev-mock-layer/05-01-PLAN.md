```typescript
// src/lib/unifi/index.ts
// Facade: re-exports real or mock UniFi client based on UNIFI_MOCK env var.
// process.env.UNIFI_MOCK is evaluated once at module initialisation (server startup),
// not per-request. Flip requires a server restart — this is intentional.
import 'server-only'

import * as real from './client'
import * as mock from './mock'

const impl = process.env.UNIFI_MOCK === 'true' ? mock : real

export const getUnifiClients          = impl.getUnifiClients
export const getFirewallPolicies      = impl.getFirewallPolicies
export const updateFirewallPolicy     = impl.updateFirewallPolicy
export const isZoneBasedFirewallEnabled = impl.isZoneBasedFirewallEnabled
```

Do NOT add any other logic. The entire file is this module switcher.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env var → module switcher | UNIFI_MOCK is read at process startup; an attacker who can set env vars has server access already |
| mock module → API responses | Mock data is hardcoded; no external input flows into mock responses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | index.ts re-exports real client when UNIFI_MOCK != 'true' | accept | Real client already protected by server-only; facade inherits same guard |
| T-05-02 | Tampering | NEXT_PUBLIC_ prefix could expose UNIFI_MOCK to client bundle | mitigate | Use unprefixed UNIFI_MOCK (no NEXT_PUBLIC_); only readable server-side |
| T-05-03 | Elevation of Privilege | Mock accidentally enabled in production if env var leaks | mitigate | UNIFI_MOCK is only set in dev.sh (not .env.production); document clearly |
</threat_model>
