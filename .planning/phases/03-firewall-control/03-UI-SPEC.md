## Interaction Patterns

### Navigation Pattern

**Top Navigation Tabs:**
- Two tabs: "Dashboard" and "Firewall"
- Active tab indicated by accent color underline or background
- Same styling as Phase 2 navigation

**Source:** D-01, D-02

### Firewall Rule Card Pattern

```
+---------------------------------------------------------------+
| [Rule Name]                              [Enabled Badge] [Switch] |
+---------------------------------------------------------------+
```

- Rule name: Left-aligned, `font-medium text-zinc-100`
- Status badge: Badge variant based on enabled state
  - Enabled: `variant="default"` (sky-600 accent)
  - Disabled: `variant="secondary"` (zinc-700)
- Switch: Right-aligned, accessible toggle

**Card styling:** Same as Phase 2 device cards
- `bg-zinc-900 border-zinc-800 rounded-lg`
- Padding: `p-4`

**Source:** D-03, D-04, D-08

### Toggle Interaction Pattern

1. **Optimistic Update (D-05):**
   - Switch animates immediately on click
   - Badge updates to show new state
   - UI shows pending state during API call

2. **Success Path:**
   - SWR revalidates data from server
   - Badge reflects final state
   - No toast needed (implicit success)

3. **Error Path (D-06):**
   - SWR `rollbackOnError: true` reverts switch to previous state
   - Toast displays: "Unable to update firewall rule. Changes reverted automatically."
   - Badge reverts to original state

**Source:** RESEARCH.md Pattern 2: SWR Optimistic Update with Rollback

## States

### Loading State
- Use `Skeleton` component from Phase 2
- Display 3-5 skeleton cards while data fetches
- Same pattern as Phase 2 device loading

### Empty State
- Icon: `ShieldOff` from lucide
- Heading: "No firewall rules found"
- Body: "Your network may not have any firewall rules configured, or there was an issue loading rules from your UniFi console."

### Error State
- Use `Alert` component with `variant="destructive"`
- Icon: `AlertCircle` from lucide
- Title: "Unable to load firewall rules"
- Body: "Please check your connection and try again."
- Action: Retry button with `RefreshCw` icon

### Offline State
- Same as Error State (API unreachable)
- Title: "Unable to reach network service"
- Body: "Please check your connection and try again."

**Source:** Patterns established in Phase 2 error-state.tsx and empty-state.tsx
