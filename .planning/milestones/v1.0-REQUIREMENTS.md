# Requirements Archive: v1.0 MVP

**Archived:** 2026-04-20
**Original defined:** 2026-04-14
**Core Value:** Visibility and control over home network traffic

## v1 Requirements — All Complete ✅

### Authentication

- [x] **AUTH-01**: User can log in with username and password — Phase 1 ✓
- [x] **AUTH-02**: User session persists across browser refreshes (7-day JWT) — Phase 1 ✓
- [x] **AUTH-03**: User can log out from any page — Phase 1 ✓
- [x] **AUTH-04**: Unauthenticated users are redirected to login page — Phase 1 ✓

### Device Visibility

- [x] **DEVI-01**: User can view all network clients with name, MAC address, and IP address — Phase 2 ✓
- [x] **DEVI-02**: User can see traffic status (High/Medium/Low/Idle) for each client — Phase 2 ✓
- [x] **DEVI-03**: User can see device name (falling back to hostname, then MAC) — Phase 2 ✓
- [x] **DEVI-04**: User can see when each client was last active — Phase 2 ✓
- [x] **DEVI-05**: Traffic data refreshes automatically (polling every 60 seconds) — Phase 2 ✓

### Device Groups

- [x] **GRUP-01**: User can create device groups with custom names — Phase 4 ✓
- [x] **GRUP-02**: User can add devices to groups — Phase 4 ✓
- [x] **GRUP-03**: User can remove devices from groups — Phase 4 ✓
- [x] **GRUP-04**: User can delete device groups — Phase 4 ✓
- [x] **GRUP-05**: Device groups persist across sessions (local storage) — Phase 4 ✓

### Group Traffic

- [x] **GTRA-01**: User can see aggregated traffic status for a device group — Phase 4 ✓
- [x] **GTRA-02**: User can see which devices in a group are currently active vs idle — Phase 4 ✓

### Firewall Control

- [x] **FWRC-01**: User can view all pre-existing firewall rules with name and enabled/disabled status — Phase 3 ✓
- [x] **FWRC-02**: User can toggle (enable/disable) a firewall rule via a simple switch — Phase 3 ✓
- [x] **FWRC-03**: Firewall rule changes are reflected immediately in the UI after toggle — Phase 3 ✓
- [x] **FWRC-04**: User sees clear error message if firewall toggle fails — Phase 3 ✓

### Historical Trends

- [x] **HIST-01**: User can view 24-hour traffic trend for the overall site — Phase 4 ✓
- [x] **HIST-02**: User can view 24-hour traffic trend for individual clients — Phase 4 ✓
- [x] **HIST-03**: Trend data shows bandwidth usage over time — Phase 4 ✓

### User Interface

- [x] **UIUX-01**: Dashboard is usable on mobile devices (responsive design) — Phase 2 ✓
- [x] **UIUX-02**: Traffic status uses clear color coding — Phase 2 ✓
- [x] **UIUX-03**: Dashboard shows "last updated" timestamp — Phase 2 ✓
- [x] **UIUX-04**: Dashboard displays meaningful error messages — Phase 1 ✓
- [x] **UIUX-05**: Dashboard shows offline/unavailable state when cloud service is down — Phase 2 ✓
