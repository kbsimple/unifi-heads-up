# Quick Task 260424-262: Fix UniFi API URLs to include consoleId in path

## What was done

Fixed all four URL constructions in `src/lib/unifi/client.ts`. The `UNIFI_CONSOLE_ID` env var was read and validated in every function but never inserted into the URL, causing every API call to hit a non-existent route and return 404.

Changed from:
```
https://api.ui.com/proxy/network/v2/api/site/default/...
```

To:
```
https://api.ui.com/ea/console/${consoleId}/proxy/network/v2/api/site/default/...
```

## Functions fixed

1. `getUnifiClients` — stat/sta endpoint
2. `isZoneBasedFirewallEnabled` — site-feature-migration endpoint
3. `getFirewallPolicies` — firewall-policies GET
4. `updateFirewallPolicy` — firewall-policies PUT

## Verification

```
grep -c "ea/console/\${consoleId}" src/lib/unifi/client.ts
# Result: 4 ✓
```
